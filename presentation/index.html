Концепт

Две подсистемы, одна цепочка доказательств

🛡️ FakeFace FinGuard

Мультимодальный анализатор: видео / аудио / текст / ссылки → ASR, OCR, извлечение сущностей, классификация сценария, детерминированный риск-скоринг.

🕸️ Shadow Graph

Граф цифровых следов: блогер → видео → сайт → промокод → Telegram → кошелёк. Находит повторяемость сущностей между источниками.

Главная ценность

Не просто «AI сказал scam», а объяснимая цепочка: видео → сайт → промокод → Telegram → повторяемость → высокий риск.

01Media Service · Студент 4

§Видео/аудио → текст

  • ffmpeg извлекает аудио (16 кГц моно) и кадры (1 кадр / 2 сек)
  • faster-whisper (small int8 / medium на GPU) — ASR аудио → transcript
  • EasyOCR (CPU, кириллица+латиница) — текст с кадров/превью (домены, промокоды)
  • Тяжёлые синхронные операции → asyncio.to_thread
  • Для звонков — детекция этапов разговора по transcript
План «Риски»: если медленно — заранее посчитанные транскрипты; OCR прогнать офлайн и отдавать из кэша.
async def process_video(path):
  wav    = extract_audio(path)   # ffmpeg
  frames = extract_frames(path)  # ffmpeg fps=0.5
  transcript = whisper(wav)     # faster-whisper
  ocr_text   = easyocr(frames)  # EasyOCR (CPU)
  return MediaResult(transcript, ocr_text)

# combined_text = transcript + ocr_text
02Entity Extraction · Студент 7

§10Сущности: regex + LLM

Проход 1 — regex (детерминированный)

URL / домены@telegram телефон KZпромокод суммы ₸/USDTBTC / ETH wallet

Телефоны и карты — только маска (4400****1234) или sha256-хэш. Правило безопасности ТЗ §0.

Проход 2 — LLM (добор)

Названия проектов/платформ, организации (eGov, КНБ, Нацбанк), pressure-фразы — то, что regex не ловит.

RE_TELEGRAM = r"@[A-Za-z0-9_]{5,32}"
RE_URL      = r"https?://[^\s]+|www\.[^\s]+"
RE_PHONE_KZ = r"(\+7|8)\s?\(?7\d{2}\)?…"
RE_PROMO    = r"(промокод|promo|код)\s*[:\-]?\s*([A-Z0-9]{4,20})"
RE_BTC      = r"\b(bc1|[13])[a-zA-HJ-NP-Z0-9]{25,39}\b"
RE_ETH      = r"0x[a-fA-F0-9]{40}"

mask_phone(p)  → "77*******67"
hash_value(p) → sha256
03Scenario Detection · vLLM

§7Классификация типа мошенничества

LLM получает combined_text и возвращает строго JSON: тип, уверенность, дополнительные сигналы и evidence-фразы.

// ответ модели
{
  "fraud_type": "fake_egov_delivery_call",
  "confidence": 0.93,
  "risk_signals": ["sms_code_request", …],
  "evidence_spans": ["назовите код из SMS"]
}
Правило надёжности: LLM не ставит risk_score — только сигналы и категорию. Скоринг детерминированный.

05Similarity Service

Поиск похожих известных scam-кейсов

  • Эмбеддинги multilingual-e5-base (ru/kz/en), размер 768
  • Коллекция scam_cases, метрика cosine
  • Payload = весь объект единого формата (label, fraud_type, entities)
  • В пайплайне: top-5 соседей
  • ≥3 соседа с label=scam и score > 0.8 → сигнал similar_to_known_scam (+20)
Скрипт src/index_dataset.py переиндексирует датасет каждый раз, когда студенты доливают разметку.
# это и сигнал, и evidence для отчёта
hits = qdrant.search("scam_cases",
        vector=embed(text), limit=5)

scam_near = [h for h in hits
  if h.score > 0.8
  and h.payload["label"] == "scam"]

if len(scam_near) >= 3:
  signals.append("similar_to_known_scam")
06Shadow Graph · Студент 8

§12Граф цифровых следов

Узлы и связи

(:Blogger)-[:PUBLISHED]->(:Video)
(:Video)-[:MENTIONS]->(:Domain)
(:Video)-[:MENTIONS]->(:Telegram)
(:Video)-[:HAS_PROMO]->(:PromoCode)
(:Call)-[:CLAIMS_AUTHORITY]->(:Org)

Телефоны/карты в граф попадают только хэшем/маской (ТЗ §0).

Главная фича демо — повторяемость

MATCH (d:Domain)<-[:MENTIONS]-(v:Video)
WITH d, count(v) AS uses
WHERE uses > 1
RETURN d.name, uses ORDER BY uses DESC

Сущность в ≥2 источниках → сигнал graph_entity_reuse (+25). Эндпоинт GET /graph/entity/{value}.

07Deepfake / Voice anomaly

Детектор синтетики в видео

  • Внешний детектор (репо fakeface-deepfake-detector) в отдельном venv
  • Конфликт версий (torch 2.6/cu124 vs основной 2.12) решён изоляцией процесса — вызов через subprocess
  • ViT — кадры лица → possible_deepfake
  • Wav2Vec2 — дорожка голоса → synthetic_voice_suspected
  • + lip_sync_anomaly — рассинхрон губ/звука
  • Срабатывает на /analyze/video и /analyze/url с deep:true
Аномалии пишутся в media_anomalies и конвертируются в риск-сигналы для скоринга.
# subprocess в изолированный venv

08OSINT · репутация доменов

Проверка извлечённых доменов

Эвристики (офлайн, без ключей)

  • Тайпсквоттинг KZ-брендов: kaspi-bonus.click, halyk-pay…, egov-…
  • Подозрительные TLD: .click .top .xyz .casino …
  • Мусорная структура хоста (длинные дефисы, цифро-буквы)

Внешний фид (опционально)

PhishTank по флагу ENABLE_PHISHTANK + ключ. По умолчанию — только эвристики.

# официальные домены → чисто
kaspi.kz            → []
# фейк под бренд → сигналы
kaspi-bonus.click   → phishing_url,
                      suspicious_domain
phishing_url suspicious_domain casino_domain_found
Модуль services/osint.py. Сигналы добавляются к доменам из entities и идут в скоринг.

09Postgres · журнал + human-in-the-loop

Каждый анализ сохраняется

Таблицы (SQLAlchemy 2.0 async + Alembic)

  • analysis_sessions — отчёт, сигналы, сущности, media_anomalies, latency, использован ли LLM

10AFM Knowledge Agent · RAG

Помощник по защите от мошенничества

Q&A по базе знаний АФМ («что делать при подозрительном звонке/сообщении»). Хранилище — та же Qdrant, отдельная коллекция afm_knowledge.

Гибридный поиск

  • dense — multilingual-e5 (семантика ru/kz/en)
  • sparse — BM25-style (точные триггер-фразы), IDF считает сам Qdrant
  • слияние — серверный RRF в Qdrant

Ответ генерирует vLLM по найденным карточкам. Без LLM — детерминированный fallback из полей карточки (агент остаётся полезным).

§0Правила безопасности данных

Что можно и что нельзя

✕ Запрещено

  • Парсить приватные Telegram-чаты
  • Скачивать/распространять реальные утечки ПДн
  • Хранить реальные карты, ИИН, SMS-коды, CVV, пароли, ЭЦП
  • Реальные записи звонков жертв без согласия
  • Выводы вида «этот человек — преступник»

✓ Разрешено

  • Открытые официальные предупреждения
  • Готовые публичные датасеты
  • Синтетические учебные звонки и посты
  • Только маскированные/хэшированные реквизиты
  • Выводы только как risk scoring low→critical
